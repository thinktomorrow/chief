AdminDesigns - Changelog v1.5

Warning - Read this before updating to version 1.5. Modifications have been made to AdminDesigns which will cause unexpected results unless you read the patch notes and understand the changes made.

General Patch Overview:

Update v1.5 is a small but useful update that adds a page template system, fixes several small bugs, and adds collapsible table rows to DataTables and FooTables. We are also going to go back and cleanup the documentation a bit. Some of the information is missing or outdated at this point and could use a refresh. All the more reason to make sure you are using the online documentation when available.

We Need Your Help!

We are trying to crack down on known bugs and theme problems. Submitting a detailed report to our support helps reduce the time it takes to discover and fix bugs. Please make use of our new helpdesk and rid those pesky bugs that slow you down. You can submit a private ticket Here, or post the problem publicly using our Bug Forum found Here.

Important: The Admin "Smooth" Skin will become depreciated in v1.55

General Development Rules:

Separating your work and customizations from the themes default styles is absolutely key to smoothly transitioning through theme patches and updates.

LESS Users:

All changes and new styles have been added to the themes base skin. That being said ALL custom changes should always be added to your child skin (default-skin/less). You should never modify the base theme as it will frequently be changed and updated. This is key to smoothly transitioning through theme patches and updates.

Standard CSS Users:

All style changes, fixes, and improvements made in theme updates will typically be added to theme.css. Users should never directly edit or modify theme.css. To prevent losing your changes, you should add all of your customizations to a new stylesheet and include it separately in your pages.

It is also highly recommended that you transition to using the themes LESS system as it is far superior, and offers a great deal of features and optimizations. A beginners Less guide has been added to the tutorials section of the documentation. It can be found here: Link

Update v1.4 - Performance and Usability Changes:

Patch v1.4 was the update of a thousand changes. Most modifications were so small they will go unnoticed by most customers. However, a thousand small changes can lead to big results. We compared the top selling admin themes on Themeforest to AdminDesigns. Read the changes below and then check out the results.

  • Documentation was entirely scrapped and recreated from the ground up. It now includes hundreds of demos covering virtually every element in the theme, including 3rd party plugins.
  • We removed hundreds if not thousands of overused HTML helper classes. Making it tremendously easier to navigate and to recreate the elements/components your application needs.
  • To help streamline development many HTML based elements will now present their source code upon hover. Click here for a Preview Image.
  • An AdminDesigns Helpdesk is now available. Our customers now have direct access to community support forums, announcements, and personal support solutions via our helpdesk ticketing system. Take a look here.
  • A major cleanup was performed on virtually every page/HTML element. We removed a great deal of leftover development code, and "hidden" elements. We're sorry they were present in the first place.
  • Several theme resources have been disabled, moved out of Core, or replaced with better/faster solutions. These changes will be the biggest nuisance as it will require customers to fix/add/remove several previous resource URLs. You can read more about these changes below.
  • Many theme/javascript resources have been better optimized, reorganized, and restructured. Resulting in drastically faster page loads and a substantial reduction in overall HTTP request. Making AdminDesigns the ONLY best selling admin theme on ThemeForest which loads in miliseconds... not seconds. Which frankly is a little scary.

Changes that can cause Breaks/Problems:

1. Glyphicons PRO icon library is no longer enabled by default
Class Description
The What: Glyphicons PRO icons have been removed from every page and theme element.
The Why: Glyphicons PRO is a huge library and should be used sparingly, or atleast not in combination with Glyphicons Halfling and FontAwesome like past updates had done.
The Fix: You should remove all Glyphicons PRO icons used in your current application. - OR - use the font library by including Glyphicons PRO on your pages.
2. PNotify is no longer the themes Core notification plugin.
Class Description
The What: PNotify has been replaced by BootBox as the themes Core notification plugin.
The Why: PNotify was a bit heavier than we wanted and we found majority of customers were using Bootbox for notifications/confirmations.
The Fix: Pages will no longer require you to link Bootbox as it is now included in Core. Pages that you want to use PNotify on will now require you to include the required plugin resources. See the plugins section of this documentation for usage.
3. All AdminPlugins have been moved into theme Core
Class Description
The What: All AdminPlugins have been minified, compressed, and moved into theme Core.
The Why: Most of these plugins were widely used and very small in size. To save on HTTP request they have been minified and merged into theme Core.
The Fix: Pages using AdminPanels, AdminDock, and AdminModals effects will no longer require you to include css/js scripts as you previously had to do. They are now activated by default.
4. Bootstrap.js has been moved into theme Core
Class Description
The What: Bootstrap.js has been minified, compressed, and moved into theme Core. assets/js/bootstrap has been removed.
The Why: The script was lightweight and used on every page. To save on HTTP request and redundant linking we have moved it to Core. One less thing to worry about.
The Fix: Any page you have included the Bootstrap.js file can be safely removed.
5. All theme Demo javascript has been moved to "js/demo"
Class Description
The What: All demo related javascript has been moved to assets/js/demo. The previous "pages" folder has been removed. Bootstrap.js has been minified, compressed, and moved into theme Core. assets/js/bootstrap. has been removed.
The Why: The previous theme javascript structure resulted in file confusion. It was unclear why and where files were used. This adds a bit of clarity.
The Fix: Most pages using demo code from assets/js/demo.js will need to have its url updated to assets/js/demo/demo.js. This also goes for most pages using the demo AdminPanel code and demo charts code.

Important Patch Info and Changes v1.4

Warning - Read this before updating to version 1.4. Large modifications have been made to AdminDesigns which will cause unexpected results unless you read the patch notes and understand the changes made.

The Biggy - Performance and Usability Changes:

Patch v1.4 was the update of a thousand changes. Most modifications were so small they will go unnoticed by most customers. However, a thousand small changes can lead to big results. We compared the top selling admin themes on Themeforest to AdminDesigns. Read the changes below and then check out the results.

  • Documentation was entirely scrapped and recreated from the ground up. It now includes hundreds of demos covering virtually every element in the theme, including 3rd party plugins.
  • We removed hundreds if not thousands of overused HTML helper classes. Making it tremendously easier to navigate and to recreate the elements/components your application needs.
  • To help streamline development many HTML based elements will now present their source code upon hover. Click here for a Preview Image.
  • An AdminDesigns Helpdesk is now available. Our customers now have direct access to community support forums, announcements, and personal support solutions via our helpdesk ticketing system. Take a look here.
  • A major cleanup was performed on virtually every page/HTML element. We removed a great deal of leftover development code, and "hidden" elements. We're sorry they were present in the first place.
  • Several theme resources have been disabled, moved out of Core, or replaced with better/faster solutions. These changes will be the biggest nuisance as it will require customers to fix/add/remove several previous resource URLs. You can read more about these changes below.
  • Many theme/javascript resources have been better optimized, reorganized, and restructured. Resulting in drastically faster page loads and a substantial reduction in overall HTTP request. Making AdminDesigns the ONLY best selling admin theme on ThemeForest which loads in miliseconds... not seconds. Which frankly is a little scary.

Speed Testing Methods and Notice:

All themes tested are current top sellers and reflect the best Themeforest has to offer. As a courtesty to our competitors we have not included the names of the theme tested. The load time of a website is based on many factors. Server performance and location will also have large impacts on load time. However, the AdminDesigns demo is currently hosted on a low cost shared hosting plan and therfor should not have an advantage over any competitor.

A Header for Migration:

So far we feel the framework is heading in the right direction and contains many of the unique features we wanted to see deployed. However, there are several items and features which could use a redesign, be better organized/explained, or should have an easier means of implementation. For this reason we have decided to take a step back from developing new features/pages and focus our attention on improving the theme in its current state.

Some of the issues we will be changing/focusing on:

  • Usability, usability, usability. Our goal in all of these changes is to vastly improve the frameworks overall usability and fluidness.
  • Documentation will be entirely reconstructed and redesigned. This is possibly our biggest priority, expect documentation size to triple.
  • Provide further ways to greatly reduce the overall size and footprint of the framework in a production enviroment. By improving existing plugin structure and adding additional LESS settings.
  • Create several video tutorials explaining some of the themes more complex topics. Particulary the frameworks LESS system and it's importantance.
  • Possibly removing the Admin Plugin folder and moving the plugins into core. As they are lightweight and frequently used
  • Review helper class usage. They feel over used in some areas.
  • To greatly improve sidebar functionality we want to convert the system into a plugin with many more settings/options
  • Make vastly better use of inline HTML comments to help explain an elements implementation and use
  • Creating a discussion forum/input portal for our customers

Migration Notices

Breaking/Ruining live sites. HTML structure sitewide will remain the same. As always we take absolutely every precaution to ensure updates have no impact on the functionality of your live sites.

General Rules:

Seperating your work and customizations from the themes default styles is absolutely key to smoothly transitioning through theme patches and updates.

LESS Users:

All changes and new styles have been added to the themes base skin. That being said ALL custom changes should always be added to your child skin (default-skin/less). You should never modify the base theme as it will frequently be changed and updated. This is key to smoothly transitioning through theme patches and updates.

Standard CSS Users:

All style changes made in update 1.2 have been added to theme.css. Users should never edit or modify theme.css. All style customizations and additions should be added to a new stylesheet and included separately in your pages.

It is also highly recommended that you transition to using the themes LESS system as it is far superior, and offers a great deal of features and optimizations. Please contact us, or leave a comment if you would like to see a LESS tutorial added to the themes documentation.
